## How to Stay Updated

Follow these steps to ensure you never miss a release.

<Steps>
  <Step title="Subscribe to Notifications" icon="bell">
    Log in to your dashboard and navigate to Account Settings > Notifications. Enable email alerts for new versions.
  </Step>
  <Step title="Follow on GitHub" icon="github">
    Visit our [GitHub repository](https://github.com/testio/platform) and watch for release events.
  </Step>
  <Step title="Check RSS Feed" icon="rss">
    Add `https://docs.example.com/rss/changelog.xml` to your feed reader.
  </Step>
</Steps>

<Tabs>
  <Tab title="Webhook Setup" icon="webhook">
    Configure webhooks to receive instant updates on new releases.

    <ParamField header="TestIO-Signature" param-type="string" required="true">
      HMAC SHA-256 signature for payload verification.
    </ParamField>

    ```javascript
    // Example Node.js webhook handler
    const crypto = require('crypto');

    app.post('/webhook/changelog', (req, res) => {
      const signature = req.headers['testio-signature'];
      const payload = JSON.stringify(req.body);
      const expectedSig = 'sha256=' + crypto.createHmac('sha256', 'YOUR_WEBHOOK_SECRET')
        .update(payload)
        .digest('hex');

      if (crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(expectedSig))) {
        console.log('New Test IO update:', req.body.version);
        res.status(200).send('OK');
      } else {
        res.status(401).send('Invalid signature');
      }
    });
    ```
  </Tab>
  <Tab title="API Polling" icon="api">
    Query the versions endpoint periodically.

    ```bash
    curl -H "Authorization: Bearer YOUR_TOKEN" \
         https://api.example.com/v1/versions?limit=5
    ```
  </Tab>
</Tabs>
